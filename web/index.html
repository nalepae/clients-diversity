Client Diversity
who proposes the blocks

Share of mainnet blocks by the software that proposed them, inferred from block graffiti.

Method

How this is measured

read shares as a floor

Each block's proposer is inferred from a small client identification segment in the block's graffiti.

Examples:

  • ssv.network BUde8dTK6c8b for Besu/Teku
  • 𝕡𝟚𝕡․𝕠𝕣𝕘 BUPR for Besu/Prysm

Blocks without the client identification are counted as unknown.

Many blocks carry no client identification, so every share is a lower bound: read it as “this client proposed at least X% of blocks”.
The true share is likely higher.

Three reasons blocks stay unidentified (see the ⓘ beside each client below):

  • Clients added or changed the client identification behaviour at a different version, some only after the Fusaka fork.
  • Among major clients, Nimbus does not implement this client identification feature yet (nimbus-eth2#7580).
  • Every client implementing this feature, except Prysm, offers a way to disable the client identification.

Conversely, a validator could deliberately set a false client identification code in its graffiti, inflating that client's apparent share, but this is very unlikely.
